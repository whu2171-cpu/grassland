# 近期 NDVI 源文件接入实验记录

日期：2026-07-06

## 背景

此前发现：

```text
D:/草地遥感/result/mask/2014-2024/maskresult/Grassland_YYYY_NDVI.tif
```

整幅影像全为空，不能作为近期 NDVI 输入。

随后检查到新的有效源文件目录：

```text
D:/草地遥感/result/1981-2024年间tiff源文件
```

其中包含：

- `2005.tif` 到 `2013.tif`
- `2014.tif` 到 `2024.tif`

经 `scripts/diagnostics/check_raster_validity.py` 检查，2014-2024 年源文件有效像元比例约为 `0.5147`，值域有效，因此可作为近期 NDVI 输入。

## 配置调整

将近期 NDVI 分组从全空的 maskresult 目录改为有效源文件目录：

```yaml
ndvi_groups:
  recent_2014_2024:
    directory: "D:/草地遥感/result/1981-2024年间tiff源文件"
    pattern: "{year}.tif"
    start_year: 2014
    end_year: 2024
```

长期 NDVI 分组仍使用：

```yaml
long_1981_2014:
  directory: "D:/草地遥感/result/tiff"
  pattern: "{year}_ndvi.tif"
  start_year: 1981
  end_year: 2014
```

其中 `1989_ndvi.tif` 缺失，构建样本时自动跳过。

## 风险边界

当前精度很高，但仍然不能作为真实草地亚型最终精度，原因包括：

1. subtype label 仍来自已有分类图和置信度筛选；
2. 近期 NDVI 与 subtype proxy 空间格局可能高度同源；
3. 即便使用 spatial block split，也不能替代外部真实 reference samples；
4. 后续必须做消融实验和独立样本验证。

因此本结果更适合表述为：

```text
近期 NDVI 能显著增强 proxy subtype 的可分性，证明植被时序背景是亚型分类的重要输入。
```

不应表述为：

```text
草地亚型真实分类精度已经达到 98%。
```

## 下一步

1. 做特征消融实验：去掉近期 NDVI、只保留长期 NDVI、只保留 ERA5、只保留地形；
2. 重新检查 SoilGrids 文件全 nodata 问题；
3. 获取或构建真实 reference samples；
4. 在真实样本验证后，再进入正式 U-Net + Transformer。

## SoilGrids 复核

后续抽查发现，`E:/1/soilgrids` 下的 `sand/silt/clay/soc/bdod` 栅格虽然文件存在，但像元值几乎全为 `-32768`，本质上仍是 nodata 文件，不能直接作为环境因子输入。

根因不是采样代码，而是早期 SoilGrids 裁剪时没有按源 VRT 的实际投影正确解释经纬度范围，导致裁剪位置落偏，输出了空壳栅格。

下一步应当先重新裁剪或重下载 SoilGrids，再把配置切回有效文件；在此之前，SoilGrids 相关特征都应暂时从模型里剔除。

1. `full`
2. `no_recent_ndvi`
3. `long_ndvi_only`
4. `era5_only`

其中各组的样本表特征列已按配置切换生效，说明特征族开关链路是通的。当前 proxy subtype 任务在 spatial block split 下的结果差异很小，主模型的随机森林仍保持 `OA = 0.9845`、`macro-F1 = 0.9878` 左右，说明现在这套 proxy label 对特征裁剪并不敏感，后续需要依赖真实 reference samples 才能做更有解释力的消融。
