# 空间划分、时序特征与地形因子扩展记录

日期：2026-07-06

## 本轮目标

在模型链路最小可运行版本基础上，完成三项优先工作：

1. 将随机验证划分改为空间 block split；
2. 将现有 2012、2013、2014 年 NDVI 栅格加入 subtype proxy 任务；
3. 基于 DEM 点位邻域计算 slope、aspect 和 TWI proxy。

本轮仍属于 proxy/internal validation，不代表真实草地亚型独立验证精度。

## 实现内容

### 空间 block split

新增 `src/classification/split.py`，统一管理训练/验证划分。

当前配置为：

```yaml
split:
  validation_fraction: 0.30
  mode: "spatial_block"
  block_size: 2.0
```

诊断结果表明，样本坐标是经纬度坐标，不能使用 `50000` 作为块大小。修正为 `2.0` 度后：

| task | 空间块数 |
|---|---:|
| fvc_ndvi | 107 |
| subtype | 102 |

### NDVI 时序特征

subtype 样本新增：

- `2012_ndvi_point`
- `2013_ndvi_point`
- `2014_ndvi_point`
- `ndvi_time_mean`
- `ndvi_time_min`
- `ndvi_time_max`
- `ndvi_time_std`
- `ndvi_time_range`

### DEM 衍生地形因子

DEM 尺寸为 `48000 x 24000`，不能整幅读入内存。因此采用点位 `3x3` 邻域窗口计算：

- `slope`
- `aspect`
- `twi_proxy`

其中 `twi_proxy` 是基于坡度的湿润指数近似项，不是基于流向和汇流累积量的正式 TWI。后续如果生成 flow accumulation，应替换为正式 TWI。

## 当前样本表字段

`proxy_samples_subtype.csv` 当前包含：

## 结果判断

1. NDVI 时序特征是当前 subtype proxy 分类提升的主要来源；
2. DEM 仍是最重要的单项地形变量；
3. 新增 `slope`、`aspect`、`twi_proxy` 后，subtype 任务在空间 block split 下小幅提升；
4. `confidence` 权重较低，说明模型主要利用地形和植被时序，而不是简单依赖伪标签置信度；
5. 类别 2 和类别 4 仍是主要风险类别，后续需要结合真实样本或参考图重点检查。

## 后续路线

下一阶段建议进入环境因子扩展：

1. 检查并接入 ERA5 降水、温度变量；
2. 检查并接入 SoilGrids 土壤有机碳、质地、容重等变量；
3. 尝试更完整的 NDVI/FVC 多年物候指标；
4. 保留 `random` 与 `spatial_block` 双验证，用于量化空间自相关影响；
5. 在特征体系和验证体系稳定后，再推进正式 U-Net + Transformer 时空协同模型。
