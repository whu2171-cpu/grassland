# 近期 NDVI 全空问题诊断与重生成方案

日期：2026-07-06

## 问题

`D:/草地遥感/result/mask/2014-2024/maskresult/Grassland_YYYY_NDVI.tif` 在 subtype 样本点采样结果全为空。该现象不合理，因此进行了只读诊断。

## 诊断结论

诊断对象：

- `Grassland_2014_NDVI.tif`
- `Grassland_2015_NDVI.tif`
- ...
- `Grassland_2024_NDVI.tif`

结果显示，这批 GeoTIFF 不是仅在 subtype 样本点为空，而是整幅影像没有任何有限值：

```text
valid_rate = 0.0
valid pixel count = 0
```

因此问题不属于样本点坐标错位或局部无覆盖，而是这批栅格文件本身已经全 NaN。

## 其他证据

`D:/草地遥感/result/mask/point0(2014-2024).csv` 和 `MODIS_NDVI_Values_Sample_Points_20140101_20240731.csv` 中存在有效 NDVI 值，说明 2014-2024 的点位 NDVI 抽样流程曾经得到过有效结果。

本地未找到 `MG_30m_YYYY.tif` 或其他 2014-2024 年 Landsat 年度原始导出栅格。因此，当前全 NaN 的 `Grassland_YYYY_NDVI.tif` 无法在本地直接修复为完整栅格。

## 判断

当前最可能的问题发生在以下阶段之一：

1. GEE 年度 NDVI 栅格导出时影像被错误 mask；
2. 本地草地掩膜后处理时把有效值全部置为 NaN；
3. nodata/NaN 写出逻辑错误；
4. 使用了错误的草地 mask 或反向 mask。

由于缺少有效原始年度栅格，本轮不建议继续尝试“修复”现有全 NaN 文件，而应重新生成。

## 重生成方案

新增 GEE 脚本：

```text
scripts/gee/export_recent_ndvi_2014_2024.js
```

策略：

1. 使用 Landsat 8/9 Collection 2 Level 2；
2. 逐年合成 2014-2024 年 5-9 月生长季 NDVI；
3. 导出未套草地掩膜的年度 NDVI；
4. 本地先检查 NDVI 栅格有效像元比例；
5. 确认有效后，再作为 `ndvi_recent_2014_2024` 分组进入模型。

这个方案避免在 GEE 导出阶段直接套草地 mask，降低再次生成全空栅格的风险。

## 本地检查工具

新增本地诊断脚本：

```text
scripts/diagnostics/check_raster_validity.py
```

下载 GEE 导出结果后，可运行：

```bash
python scripts/diagnostics/check_raster_validity.py <导出目录> --pattern "Recent_NDVI_*.tif"
```

若 `valid_rate > 0` 且值域大致在 `-1` 到 `1` 之间，再把这些文件接入配置。
