# 汇报准备：模型先行与数据同步方案

日期：2026-07-06

## 1. 汇报主线

当前研究的核心思路应表述为：

> 以时空特征协同提取模型为主线，先用现有 FVC/NDVI 分类产品和高置信伪标签跑通模型构建、训练验证、分类制图、变化检测和驱动解释链路；同时并行收集草地亚型地面真值和权威参考图，待真值数据补齐后，将占位标签替换为草地亚型标签，形成最终的高精度草地植被亚型分类与变化机制研究。

这个策略的关键是把“FVC 分类”定位为模型链路的代理任务，而不是最终研究结论。

建议汇报时使用以下表述：

- 可以说：目前先以 FVC/NDVI 分类或现有高置信亚型图作为 proxy label，用于模型结构验证和流程打通。
- 可以说：最终论文目标仍然是草地植被亚型分类，不会把 FVC 分类精度等同于亚型分类精度。
- 不建议说：已经完成草地亚型地面真值验证。
- 不建议说：FVC 分类结果就是草地亚型分类结果。

## 2. 目前整体方案

### 2.1 总体技术路线

1. 数据准备与预处理；
2. 多源输入与样本构建；
3. 时空深度模型构建；
4. 高精度分类产品生成；
5. 精度验证与模型优化；
6. 亚型变化检测；
7. 驱动机制分析。

其中主模型是：

> Sentinel-2 时序影像 + 辅助环境因子 -> U-Net 空间特征提取 -> Transformer 时间特征提取 -> 时空特征融合 -> 草地植被亚型分类。

辅助模块是：

- ARNC：物候解释、样本辅助、轻量 baseline；
- DT/RF/XGBoost：传统模型对照；
- FVC/NDVI 分类：模型链路先行阶段的 proxy task；
- GeoDetector/SHAP/SEM：后续变化机制解释。

### 2.2 为什么可以先用 FVC 分类跑模型

目前最大限制不是模型框架，而是高质量草地亚型地面真值不足。若等全部真值数据齐备后再搭模型，会导致主线进展滞后。因此可以采用“两阶段标签策略”：

第一阶段：代理标签阶段。

- 使用 FVC/NDVI 分类、现有亚型图高置信像元或生态状态图作为 proxy label；
- 目标是验证数据输入、patch 构建、时序编码、空间编码、融合模块、训练流程、输出图件和变化检测是否可运行；
- 输出结果只作为工程验证和方法预实验。

第二阶段：真值替换阶段。

- 获取 2009 年内蒙古草地类型调查产品、1980s 中国植被类型图、野外样方或人工解译样点；
- 将 proxy label 替换或校正为草地亚型 reference samples；
- 重新训练主模型并进行正式精度评价；
- 最终报告 OA、PA、UA、F1、macro-F1、空间外推验证和不确定性。

这样做的优点：

- 模型结构和代码链路可以立即推进；
- 数据收集不阻塞模型开发；
- 后续拿到真值后只需替换样本层和标签层，不需要重建整个工程；
- 明天汇报时逻辑更稳：问题被识别，解决路径也清楚。

## 5. 后续工作计划

### 第 1 周：模型链路最小可运行版本

目标：先跑通模型工程链路。

任务：

1. 确定 proxy label：
   - 方案 A：FVC/NDVI 分级分类；
   - 方案 B：`Subtype_Classification_Full.tif` 高置信伪标签；
   - 方案 C：GMM 生态状态图。
2. 构建统一输入网格；
3. 生成训练 patch 和时序样本；
4. 搭建 U-Net 空间分支和轻量 Transformer 时间分支；
5. 输出第一版分类图和内部验证指标。

交付：

- 最小训练脚本；
- 配置文件；
- 一张流程图；
- 一张 proxy 分类结果图；
- 一张模型结构图。

### 第 2 周：加入环境因子与对照模型

任务：

1. 加入 DEM、坡度、ERA5、SoilGrids 等辅助因子；
2. 比较 U-Net only、Transformer only、U-Net + Transformer；
3. 加入 RF/XGBoost 传统模型对照；
4. 输出初步消融实验。

交付：

- 模型对比表；
- 消融实验表；
- 误分类区域图；
- 不确定性图。

### 第 3-4 周：真值数据替换与正式验证准备

任务：

1. 根据老师反馈获取草地类型图或样方；
2. 若无法获得，建立 100-200 个人工解译点；
3. 替换 proxy label，形成 reference samples；
4. 做空间分块交叉验证；
5. 输出正式精度评价。

交付：

- reference samples 表；
- 混淆矩阵；
- OA/PA/UA/F1/macro-F1；
- 空间分块验证结果。

### 后续阶段：变化检测与机制解释

任务：

1. 基于多期分类结果生成转移矩阵；
2. 构建草地亚型退化/恢复方向；
3. 将转移强度作为因变量，回接 GeoDetector、SHAP 或 SEM；
4. 解释气候、地形、土壤和人类活动对亚型变化的贡献。

## 6. 汇报中的一句话结论

建议最后用这句话收束：

> 当前研究已经明确了“模型先行、数据同步”的推进策略：先用 FVC/NDVI 分类和高置信伪标签跑通 U-Net + Transformer 时空协同模型及变化分析链路，同时通过老师和合作资源补齐草地亚型地面真值。待权威样本或参考图获取后，将代理标签替换为草地亚型真值，完成正式精度验证和机制解释。

## 7. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 草地亚型真值数据短期拿不到 | 无法形成正式精度评价 | 先用 proxy task 跑通模型；同步建立人工解译验证点 |
| FVC 分类与草地亚型不完全一致 | 代理任务可能偏离最终任务 | 汇报中明确 FVC 只是工程链路验证，不作为最终亚型结论 |
| 深度模型训练样本不足 | 模型过拟合或不稳定 | 高置信伪标签 + 空间分块验证 + 传统模型对照 |
| 多源数据分辨率不一致 | 输入对齐和尺度效应问题 | 统一网格、记录重采样方式、做尺度敏感性分析 |
| 随机划分导致精度虚高 | 验证结果不可信 | 使用空间 block split 或区域外推验证 |
